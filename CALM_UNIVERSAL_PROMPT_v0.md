§4 — THE PER-SUMMIT DELIVERABLE SHAPE

For every bag, ship ALL of:

  1. The artifact at its canonical path (per the route map's hint or, if
     ambiguous, a sibling of the closest existing doc).
  2. A gate script at ~/CredexAI/scripts/everest_NN_zkbb_<slug>_gate.py
     that exits 0 = green / 1 = red and prints PASS/FAIL lines.
  3. Tests where applicable (≥ enough that the gate's pytest-passed
     assertion is meaningful).
  4. A status-line update on the matching summit in the route map,
     prefix `*Status:* **BAGGED (Summit NN/300) YYYY-MM-DD** — <evidence>`.
  5. A chained `summit_bagged` record in ~/.calm-vault/user_state.jsonl
     (per the convention in USER_STATE_PROTOCOL.md). The parallel
     auto-logger handles this if you announce the bag in your reply;
     if not present, write it yourself.
  6. A one-line task in TaskCreate with the summit number(s).

If your bag is documentation-only (governance, audit, scope), the gate
verifies presence + key-phrase content. If it's code, the gate runs
pytest. If it's an institutional artifact (MoU, signed agreement), the
gate verifies the signed PDF / committed text is present at the path.
