# Everest 70 — Replay Defense

*Phase VIII — Counterparty & Transport Security. Prereq: Everest 65, 66, 67, 69.*

## Acceptance Criterion

A nonce generated by C and bound into the proof prevents replay: the same proof cannot be replayed by any party to a different verifier or in the same session. The nonce must be:

- **C-supplied:** the counterparty (C) generates a fresh 32-byte cryptographically random nonce per request.
- **Bound into the FS transcript:** the nonce is incorporated into the Fiat-Shamir challenge computation so tampering with the nonce flips the entire proof.
- **Fresh-checked on verify:** the verifier maintains a minimal replay cache (24-hour sliding window by default) and rejects any nonce that appears a second time.

## Threat Model

### C-Relay Attack
A malicious C2 observes a disclosure proof issued to C1. C2 replays the same proof to a third party. **Mitigation:** The third party uses a different nonce, so the challenge bound to C's nonce is different; the replayed proof's challenge does not match. Rejected.

### O-Replay Attack
A malicious operator (O) caches an old proof and re-issues it to the same C on a later request. **Mitigation:** The later request contains a fresh nonce; the old proof's challenge was computed over the earlier nonce. Challenge mismatch. Rejected.

### Cross-Session Replay
A proof from session S1 is reused in session S2. **Mitigation:** Each session uses a fresh nonce; the proof is bound to S1's nonce and cannot satisfy S2's challenge. Rejected.

### Cross-Predicate Replay
A proof for predicate p1 is reused for predicate p2. **Mitigation:** This is already defended by Everest 65 (predicate_id is in the FS transcript); Everest 70 adds nonce binding as an orthogonal layer.

## Verification Rules

When `verify_disclosure()` is called:

1. **Extract nonce:** From the DisclosureResponse, extract the `challenge_nonce_hex` field and decode to `nonce_bytes`.
2. **Check length:** Reject if nonce is not exactly 32 bytes.
3. **Replay check:** Call `replay_cache.seen(nonce_bytes, time.time())`. If `True`, reject with reason `nonce_replay`.
4. **Context binding:** The `build_context()` function already incorporates the nonce; verifying the sigma proof implicitly verifies that the nonce was not tampered.
5. **Standard checks:** Proceed with all other verification steps (operator sig, predicate_id match, chain head match, etc.).

## Implementation Notes

### Memory Bounds

The replay cache is bounded by:
- **Default TTL:** 24 hours.
- **Typical entry size:** 32 bytes (nonce) + 8 bytes (timestamp) = 40 bytes per entry.
- **Typical load:** If a principal discloses one predicate per second for 24 hours, ~86,400 entries → ~3.5 MB.
- **Hard limit:** Implementations MAY enforce a cap (e.g., 1M entries) and fail closed when exceeded.
