# Everest 90 — Third-Party Security Audit Prep

*Phase VII — Engineering Reliability. Prereq: Everests 81 (Rust Production Implementation), 85 (Adversarial Fuzzers).*

---

## What this summit ships

An audit-ready packet of artifacts that a reputable third-party security audit firm (Trail of Bits, NCC Group, Cure53, Doyensec) can pick up cold and execute against. The packet is what makes the audit *startable* without back-channel onboarding — the actual audit run is downstream and tracked separately.

This summit's deliverable is the **packet**. The audit itself, and its remediation, are operational milestones that follow.

---

## Audit packet contents

The packet is a tarball + an index document. Eleven items:

1. **Protocol spec.** `ZKBB_USER_PROTOCOL_v0.md` + all referenced summit deliverables (everests/).
2. **Threat model.** Consolidated from Everests 1, 9 (Failure-Mode Catalogue), 21 (Enrollment Fraud Taxonomy), 23 (Recovery From Enrollment Loss), 94 (Rubber-Hose Envelope).
3. **Cryptographic spec.** Everests 44 (Pedersen), 45 (Range Proof), 46 (Template Commitment), 48 (Cross-Template Consistency Proof), 65 (Predicate ZK Generator), 96 (Post-Quantum Migration).
4. **Code.** `calm-witness` Rust crate at the audit-frozen commit; SHA-pinned dependency tree; Cargo.lock attached.
5. **SBOM.** Full software bill of materials in SPDX format. Generated by `cargo cyclonedx` + manual review.
6. **Conformance vectors.** All published vector sets across Everests 36, 37, 44, 45, 65 — ~10 000 test vectors.
7. **Fuzzer history.** Last 90 days of fuzzer corpus + any historical crashes (with severity classification + fix commits).
8. **Prior bug history.** Internal-found bugs since project inception, with severity + fix commit + lessons-learned write-ups.
9. **Architecture diagrams.** Trust boundaries (Everest 10), data flows, key custody, vault encryption topology.
10. **Test access.** Live test infrastructure (Sigsum dev log endpoint, Roughtime dev anchor, CredexAI dev issuer, dev-mode operator binary) for the auditor to exercise without affecting production.
11. **Designated point of contact.** Calm Foundation engineer available within 24 hours of an auditor question during the audit window; published rotation schedule.

## Audit scope (recommended)

**In scope:**
- Cryptographic correctness of Pedersen + Bulletproofs + Schnorr-sigma compositions.
- Side-channel resistance (timing, memory access patterns, allocator behavior).
- Memory safety (Rust's safety + audit of `unsafe` blocks).
- Fuzzer coverage assessment (does the fuzzer corpus actually exercise the attack surface?).
- Consent calculus implementation (Everests 71-78, 8).
- Biometric pipeline privacy claims (do features ever leak? does audio ever persist?).
- Anti-replay defense (Everests 49, 50, 70).
- Threat-model coverage gaps.

**Out of scope:**
- Behavioral biometric science (forensic-handwriting / lexical-fingerprint FAR/FRR — separate Everest 40 study with academic partners).
- Governance / legal review (separate engagement).
- Production-deployment operational risk (separate engagement with operational consulting).

## Engagement terms (recommended for the contract)

- **Black-box-first.** Auditor begins with the packet only; spec-clarification questions logged.
- **Then white-box.** After 1 week, auditor moves to source-with-context. Foundation engineer joins for source-level walk-throughs.
- **Open report.** The audit report is published openly, with Foundation responses inline. Critical / high findings include the fix commit hash.
- **No NDA on findings.** Auditor's findings are publishable from day one. Foundation may request a 30-day delay only for unmitigated critical issues, with a hard ceiling of 90 days.

These terms are non-standard for security audits; the openness premium is what makes Calm Witness a credible open standard.

## Acceptance test

**T-90.1 (packet exists).** All 11 packet items present as a single tarball + an index doc.
**T-90.2 (auditor cold-start).** A pilot auditor (internal volunteer or vetted external) starts the audit using only the packet; logs questions during the first 48 hours; answers from the index doc resolve ≥ 80% of questions.
**T-90.3 (SBOM correctness).** SBOM matches the actual dependency tree of the audit-frozen commit (verified via `cargo cyclonedx --output-format json` regenerated and compared).
**T-90.4 (live test infra working).** Auditor exercises the dev infrastructure end-to-end within the first 4 hours: enrollment → disclosure → verification.
**T-90.5 (publicly-committed remediation policy).** Calm Foundation publishes its commitment to fix all "critical" and "high" findings before v1.0, with a documented remediation timeline.
**T-90.6 (auditor candidate list ready).** At least 3 reputable audit firms identified, scoped, and quoted; one selected for the first engagement.
