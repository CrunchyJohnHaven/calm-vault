A credential broker
for AI agents.

Your agents should never hold your keys. Calm Vault stores secrets locally, encrypted with your passphrase, and hands out short-lived, scoped, revocable grants instead. One file. One passphrase. No cloud.

# 3. issue a 60-second grant scoped to slack-bot
$ calm-vault grant slack-token --agent slack-bot --duration 60

# 4. the agent redeems it, just-in-time
$ calm-vault request slack-token --grant "$GRANT"

# 5. revoke anything, by grant id / agent / credential
$ calm-vault revoke slack-bot

How it works

Three boring ideas, executed honestly.

🔐

Local root of trust

The master key is derived from your passphrase via Scrypt (n=2¹⁵, r=8, p=1). The vault file is Fernet-encrypted at rest — AES-128-CBC plus HMAC-SHA256. No service to call. No account to sign up for.

⏱️

Short-lived grants

Every grant is an HMAC-signed envelope with a credential id, agent name, issue time, expiry, and nonce. Default duration is five minutes. The agent never sees the master key — just a one-shot ticket.

🪓

Real revocation

Kill a grant by id, an agent by name, or a credential by name. Revocation is immediate and cascades — if you revoke a credential, every outstanding grant for it stops working on the next redemption.

📜

Append-only audit

Every setup, issue-agent, add, grant, request, and revoke writes a JSON line to audit.log. No plaintext values — just operations, timestamps, and identifiers. Replayable forever.

🧱

One file, no daemon

The whole broker is one Python file, ~450 lines, that you can read in twenty minutes. No background process. No service to keep alive. Wrap it in your agent harness as a subprocess and you're done.

🎁

Apache 2.0, no telemetry

Fork it, embed it, ship it inside your product. Calm Vault has zero outbound network calls and zero analytics. The only thing it knows about you is what you typed into it.

Threat model (the short version)

Honest scope. V1 is the small, local, honest version — bigger guarantees are on the roadmap.

Calm Vault protects against

  • Agents leaking credentials they were never given.
  • Long-lived agent compromise — grants expire in minutes.
  • Replay after revocation — revoked grants are rejected.
  • Tampering with grants — every grant is HMAC-signed.
  • Casual disk inspection — vault data is Fernet-encrypted at rest.

Calm Vault does not protect against

  • A passphrase you typed into the agent's context.
  • A compromised machine with the vault already unlocked.
  • An agent that pipes the redeemed value into a prompt-replayable scratchpad.
  • Side channels — V1 is single-machine, process-level.

What's coming

V1 ships today. V2 closes the loop.

calm-vault serve

Daemon mode with a Unix-socket API so agents don't have to spawn the CLI on every redemption.

Auto-refreshing issuers

OAuth-style upstream token refresh — Calm Vault rotates the underlying credential before grants are minted.

Key rotation

Cosign-style master-key rotation and one-shot re-encryption without losing audit history.

Pluggable backends

1Password, AWS Secrets Manager, HashiCorp Vault — Calm Vault as the agent-facing facade, your existing vault as storage.

Hardware-backed keys

TPM, Secure Enclave, YubiKey FIDO2 PRF. The passphrase becomes a second factor, not the only factor.

Multi-agent policies

Per-agent quotas, time-of-day windows, credential allowlists. Defense in depth for fleet operators.
