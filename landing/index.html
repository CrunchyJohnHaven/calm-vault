The problem

You let an AI agent operate your business. It needs your Stripe key, your Anthropic token, your Coinbase wallet, your GitHub PAT, and 10–50 more credentials, each scary. The market gives you three bad options.

📄

Plaintext files

The agent reads them whenever it wants. No revoke. No audit. No protection.

🔑

1Password Service Account

Designed for human teams. $7.99/user/month. No per-use biometric on API. Closed source.

🏰

HashiCorp Vault

Production-grade, but 100+ ops hours to set up. Built for Kubernetes, not for a founder with multiple AI instances.

The solution

Calm Vault treats your AI agent as a cryptographic holder. You issue signed grants per credential, per use. The agent presents the grant; the broker verifies; the credential is released — once, for that nonce, for that agent, for that brief window. When you're done, you revoke.

# John issues a 5-min grant for the Stripe key $ calm-vault grant stripe-live --duration 300 # prints a signed JSON grant — pass to the agent # Agent uses the grant once $ calm-vault request stripe-live --grant '...' # prints the credential to stdout # Same grant used again — DENIED $ calm-vault request stripe-live --grant '...' # → ACCESS_DENIED_NONCE_REPLAY # Nuclear: stop everything immediately $ calm-vault revoke-all # → all future access denied
