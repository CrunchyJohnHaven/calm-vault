# Cross-Primitive Envelope Format v0 — Calm ZKAC Substrate

**Draft v0 · 2026-05-20 · Calm**
**Closes Everest 122 of [`ZKAC_NEXT_200_EVERESTS.md`](ZKAC_NEXT_200_EVERESTS.md).**
**Composition:** [`CALM_PACT_PROTOCOL_v0.md`](CALM_PACT_PROTOCOL_v0.md), [`CALM_WITNESS_WIRE_FORMAT_v0.md`](CALM_WITNESS_WIRE_FORMAT_v0.md), [`CALM_COMPASS_PROTOCOL_v0.md`](CALM_COMPASS_PROTOCOL_v0.md).

## §1 — Purpose and scope

A counterparty asks an operator to disclose one or more primitive bits across the Calm suite: a Pact equality proof (directive alignment), one or more Witness state predicates (principal baseline, biometric match, unusual mental state), and/or one or more Compass values predicates (unselfish behavior, cross-tribal engagement, respect for difference, no known willful harm). The operator responds with a **single composite envelope** that carries all requested primitives, omits all unrequested ones, and binds them under a unified signature. The counterparty learns only what it asked for; cardinality is hidden (a principal disclosing one value and a principal disclosing ten values both produce the same-shaped envelope).

This format is the substrate for multi-primitive disclosure: it allows Pact, Witness, and Compass to compose without leaking the number of disclosures or revealing unrequested primitives to verifiers or intermediaries.

## §2 — Top-level envelope shape

```json
{
  "wire_version": "calm-cross-primitive-envelope/v0",
  "kind": "CrossPrimitiveEnvelope",
  "envelope_id": "<64-hex UUID>",
  "session_nonce": "<string>",
  "request_digest": "<64-hex SHA-256>",
  "principal_did": "<CredexAI DID>",
  "counterparty_did": "<CredexAI DID>",
  "chain_head": "<64-hex chain-state hash>",
  "issued_at_iso": "<ISO 8601>",
  "issued_by_operator": "<64-hex Ed25519 pubkey fingerprint>",
  "primitive_sections": {
    "pact": null | { ... PactSection ... },
    "witness": null | { ... WitnessSection ... },
    "compass": null | { ... CompassSection ... }
  },
  "refusal_metadata": {
    "refusal_reason": null | "<string>",
    "refusals_by_primitive": {
      "pact": null | "<reason>",
      "witness": null | "<reason>",
      "compass": null | "<reason>"
    }
  },
  "operator_signature": "ed25519:<hex>"
}
```

**Field semantics:**

- `wire_version`: Fixed string `"calm-cross-primitive-envelope/v0"`. Implementations MUST reject unknown versions.
- `kind`: Fixed string `"CrossPrimitiveEnvelope"`.
- `envelope_id`: A collision-resistant UUID (v4 or content-addressed hash) generated by the operator; stable across the request-response lifecycle.
- `session_nonce`: Copied from the counterparty's request; binds the envelope to the originating session.
- `request_digest`: SHA-256(canonical JSON of request dict, minus counterparty signature). Allows the operator to prove it received the specific request this envelope is answering.
- `principal_did`: CredexAI-issued DID of the principal (the human operator is attesting about).
- `counterparty_did`: CredexAI-issued DID of the counterparty (who asked the question).
- `chain_head`: The principal's vault chain head hash at disclosure time. Binds the predicates to a specific chain state.
- `issued_at_iso`: ISO 8601 timestamp of envelope generation.
- `issued_by_operator`: Ed25519 public-key fingerprint of the operator's signing key.
- `primitive_sections`: A dict with three keys: `pact`, `witness`, `compass`. Each is either `null` (not requested) or an object (requested and disclosed, subject to consent).
- `refusal_metadata`: If any primitive was refused (consent not granted, predicate failed, etc.), the refusal reason goes here per-primitive. Absence of a reason string means the primitive succeeded.
- `operator_signature`: Ed25519 signature of the canonical JSON of the envelope minus this field. Covers the entire envelope content, including the refusal metadata.

## §3 — PactSection shape

```json
{
  "wire_version": "calm-pact/v0.1",
  "commitments": {
    "principal_commitment": "<hex Ristretto255 group element>",
    "counterparty_commitment": "<hex Ristretto255 group element>"
  },
  "proof": {
    "delta_group_element": "<hex>",
    "schnorr_proof": {
      "a": "<hex>",
      "e": "<hex>",
      "z": "<hex>"
    }
  },
  "categorical_depth": <int>,
  "alignment_bit": true | false
}
```

**Field semantics:**

- `wire_version`: Indicates the Pact cryptographic version baked into this section (e.g., `"calm-pact/v0.1"` for Ristretto255).
- `commitments.principal_commitment`: The principal's Pedersen commitment to its directive (Calm Pact §4.2 step 1).
- `commitments.counterparty_commitment`: The counterparty's commitment (Calm Pact §4.2 step 2). Echoed by operator for round-trip verification.
- `proof.delta_group_element`: The delta = C_A / C_B computed by the operator per Calm Pact §4.2 step 3.
- `proof.schnorr_proof`: The Σ-protocol proof of knowledge that delta is a power of h (i.e., principal and counterparty have the same directive).
  - `a`: The commitment in the first round of the Σ-protocol.
  - `e`: The challenge (Fiat-Shamir hash of a, delta, and context).
  - `z`: The response scalar.
- `categorical_depth`: The tree depth to which alignment was checked (e.g., depth 2 for "same intervention class but possibly different specific tactic"). Determines how granular the alignment is.
- `alignment_bit`: Boolean result. `true` iff the Schnorr proof verified and directives are aligned to the requested depth.

If the counterparty did NOT request Pact disclosure, the entire `pact` section is `null`.

## §4 — WitnessSection shape

```json
{
  "wire_version": "calm-witness/wire/v0",
  "disclosures": [
    {
      "predicate_id": "cwp.v0.in_baseline_24h",
      "commitment_hex": "<hex>",
      "proof": {
        "a0": "<hex>", "a1": "<hex>",
        "e0": "<hex>", "e1": "<hex>",
        "z0": "<hex>", "z1": "<hex>",
        "claimed_bit": 0 | 1
      }
    },
    ...
  ],
  "consent_check": true
}
```

**Field semantics:**

- `wire_version`: Fixed to `"calm-witness/wire/v0"`.
- `disclosures`: Array of predicate disclosures, one per requested Witness predicate. Each is a `PredicateDisclosure` (Calm Witness Wire Format §5). Only predicates the counterparty requested appear here.
- `consent_check`: Boolean confirming that the operator verified `principal_consents_to_disclose(predicate_id, counterparty_class)` for every disclosure in this array. Verifiers MUST treat `false` as a protocol violation.

If Witness was not requested, the entire `witness` section is `null`.

## §5 — CompassSection shape

```json
{
  "wire_version": "calm-compass/v0",
  "disclosures": [
    {
      "predicate_id": "cwp.compass.v0.unselfish_act_in_window_30d",
      "aggregate_commitment": "<hex Pedersen commitment>",
      "range_proof": {
        "commitments": [ "<hex>", ... ],
        "responses": [ "<hex>", ... ]
      },
      "classifier_hash": "<sha256:hex>",
      "claimed_bit": true | false,
      "dispute_status": null | "disputed" | "unknown"
    },
    ...
  ],
  "chain_head_binding": "<64-hex>",
  "consent_check": true
}
```

**Field semantics:**

- `wire_version`: Fixed to `"calm-compass/v0"`.
- `disclosures`: Array of values-predicate disclosures, one per requested Compass predicate. Only requested predicates appear.
  - `predicate_id`: Compass predicate ID (e.g., `"cwp.compass.v0.unselfish_act_in_window_30d"`).
  - `aggregate_commitment`: Pedersen commitment to the aggregate of predicate evaluations over the principal's chain (Calm Compass §4).
  - `range_proof`: Bulletproof that proves the aggregate count exceeds the threshold without revealing the count. Wire format per the Compass reference implementation.
  - `classifier_hash`: SHA-256 of the evaluator function source code. Allows any verifier to confirm which logic was used to compute the predicate.
  - `claimed_bit`: Boolean result. `true` iff the predicate evaluated as satisfying the principal's criteria within the time window.
  - `dispute_status`: For the `no_known_willful_harm` predicate specifically, can be `null` (no disputes), `"disputed"` (active counter-claim exists), or `"unknown"` (no evidence records). For other predicates, always `null`.
- `chain_head_binding`: The principal's vault chain head at evaluation time. Binds the entire Compass section to a specific chain state, matching the envelope's top-level `chain_head`.
- `consent_check`: Boolean confirming operator verified `principal_consents_to_disclose(predicate_id, counterparty_class)` for all Compass predicates.

If Compass was not requested, the entire `compass` section is `null`.

## §6 — Selective disclosure enforcement

A verifier receiving a CrossPrimitiveEnvelope MUST enforce the following rules:

1. **Unrequested sections are forbidden.** If the original request asked for `[Witness.in_baseline_24h]` only, the `pact` and `compass` sections in the response MUST both be `null`. If either is non-null, the verifier MUST reject the entire envelope with a "smuggled primitive" error.

2. **Cardinality privacy.** The envelope structure is identical whether the principal disclosed 1 or 10 predicates across all three primitives. An observer examining just the envelope cannot tell how many disclosures are inside without deserializing the `primitive_sections`. Verifiers treat a `null` section as "not present" and do not count it toward cardinality.

3. **Refusal floor inheritance.** Calm Witness has a refusal floor (§4 of PREDICATE_VOCABULARY_v0.md) and Calm Compass has a refusal floor (§4 of COMPASS_PREDICATES_v0.md). If any requested predicate violates its primitive's refusal floor (e.g., requesting a predicate that would disclose medical diagnosis), the operator refuses at predicate-evaluation time, and the entire envelope is rejected. The envelope MUST carry a `refusal_reason` explaining why. Verifiers treat this as a protocol-compliant refusal, not a tampering attempt.

4. **Per-predicate verification.** Each section's disclosures are verified using the corresponding primitive's verifier:
   - Pact proofs verify via the Schnorr equality proof (Calm Pact §4.2).
   - Witness proofs verify via the Σ-protocol disjunction proof (Calm Witness §4 and §8).
   - Compass proofs verify via the range proof + classifier-hash check (Calm Compass §4).

   A failure in any primitive's verification rejects the entire envelope.

## §7 — Wire-level integrity and replay detection

**Outer signature binding:**

The `operator_signature` field covers the entire envelope content (all fields except `operator_signature` itself) serialized as canonical JSON (sorted keys, compact separators, no whitespace). Two independent signatures are NOT present; there is a single outer signature binding all three primitives.

```
operator_signature covers:
  wire_version, kind, envelope_id, session_nonce,
  request_digest, principal_did, counterparty_did,
  chain_head, issued_at_iso, issued_by_operator,
  primitive_sections, refusal_metadata
```

**Replay detection:**

An attacker cannot replay an envelope across sessions because:

1. The `session_nonce` from the request is baked into the envelope.
2. The `request_digest` binds the envelope to the specific request that was issued.
3. The operator's signature covers both of these fields.

If an attacker tries to replay envelope E from session 1 to session 2 (with a different nonce), the replay will be detected by verifiers because the nonce has changed and the signature no longer covers it.

## §8 — Worked example

A counterparty asks:

```
Request: {
  "kind": "CrossPrimitiveDisclosureRequest",
  "counterparty_id": "did:example:counterparty",
  "session_nonce": "nonce_abc123",
  "requested_primitives": {
    "pact": { "categorical_depth": 2 },
    "witness": { "predicate_ids": ["cwp.v0.in_baseline_24h"] },
    "compass": { "predicate_ids": ["cwp.compass.v0.no_known_willful_harm_in_window_365d"] }
  },
  "chain_head_at_request": "cafe...",
  "counterparty_signature": "ed25519:..."
}
```

Operator receives request, verifies signature, computes `request_digest` = SHA-256(request sans signature).

Operator runs three parallel evaluations:

1. **Pact:** Computes `delta` from received commitments, Schnorr proof, checks equality to depth 2. Result: `alignment_bit = true`.
2. **Witness:** Evaluates `cwp.v0.in_baseline_24h`, verifies consent, computes Σ-protocol proof. Result: principal was in baseline in last 24h, bit = `true`.
3. **Compass:** Evaluates `cwp.compass.v0.no_known_willful_harm_in_window_365d`, computes range proof, checks for active counter-claims. Result: no active counter-claims, `dispute_status = null`, bit = `true`.

Operator mints envelope:

```json
{
  "wire_version": "calm-cross-primitive-envelope/v0",
  "kind": "CrossPrimitiveEnvelope",
  "envelope_id": "uuid-1234",
  "session_nonce": "nonce_abc123",
  "request_digest": "cafe...",
  "principal_did": "did:example:principal",
  "counterparty_did": "did:example:counterparty",
  "chain_head": "deadbeef...",
  "issued_at_iso": "2026-05-20T15:30:45Z",
  "issued_by_operator": "ed25519:...",
  "primitive_sections": {
    "pact": {
      "wire_version": "calm-pact/v0.1",
      "commitments": { ... },
      "proof": { ... },
      "categorical_depth": 2,
      "alignment_bit": true
    },
    "witness": {
      "wire_version": "calm-witness/wire/v0",
      "disclosures": [
        {
          "predicate_id": "cwp.v0.in_baseline_24h",
          "commitment_hex": "...",
          "proof": { ... }
        }
      ],
      "consent_check": true
    },
    "compass": {
      "wire_version": "calm-compass/v0",
      "disclosures": [
        {
          "predicate_id": "cwp.compass.v0.no_known_willful_harm_in_window_365d",
          "aggregate_commitment": "...",
          "range_proof": { ... },
          "classifier_hash": "sha256:...",
          "claimed_bit": true,
          "dispute_status": null
        }
      ],
      "chain_head_binding": "deadbeef...",
      "consent_check": true
    }
  },
  "refusal_metadata": {
    "refusal_reason": null,
    "refusals_by_primitive": {
      "pact": null,
      "witness": null,
      "compass": null
    }
  },
  "operator_signature": "ed25519:..."
}
```

Verifier receives envelope:

1. Checks `wire_version` is `"calm-cross-primitive-envelope/v0"`.
2. Verifies `operator_signature` covers all top-level fields.
3. Validates that only requested primitives are non-null (Pact, Witness, Compass all requested; all three non-null = pass).
4. Verifies Pact Schnorr proof.
5. Verifies Witness Σ-protocol proof against the commitment.
6. Verifies Compass range proof + classifier hash match known evaluators.
7. Checks `consent_check` is `true` for each section.
8. Checks `refusal_metadata.refusal_reason` is `null` (no refusals).
9. Returns a structured result: `{pact: true, witness: true, compass: true}` to the counterparty's application logic.

## §9 — Tampered-section rejection

**Scenario:** An attacker intercepts the envelope and flips the Compass `claimed_bit` from `true` to `false`.

**Outcome:** The operator's signature no longer covers the modified `primitive_sections`. The verifier recomputes the canonical JSON of the entire envelope (minus signature) and verifies the signature using the operator's public key. The verification fails, and the envelope is rejected as tampered.

**Alternative scenario:** Attacker deletes the Witness section (sets it to `null`) even though Witness was requested.

**Outcome:** The verifier checks the request to confirm Witness was requested. The request says it was; the envelope says it was not (null). The verifier treats this as a "missing-required-primitive" error and rejects the envelope.

## §10 — Falsifiability section

The cross-primitive envelope format is falsifiable if, and only if:

1. **Proofs are verifiable by an independent implementation.** A second implementation of the verifier (in a different language, by a different team) must be able to verify envelopes produced by the reference implementation, and reject the same mutations the reference rejects. This is guaranteed by the use of standard cryptographic components (Pedersen commitments, Σ-protocols, Ed25519, Bulletproofs) with published specifications.

2. **Selective disclosure is provably enforced.** An envelope claiming to disclose only Witness predicates but containing a Compass section violates the format. The format is falsifiable if we can produce such an invalid envelope and show that a correct verifier rejects it.

3. **Chain binding is cryptographically verifiable.** The `chain_head` field must match the principal's actual vault chain head at disclosure time. If an operator mints an envelope with a fake chain head, the principal can produce a counter-proof from their chain showing the discrepancy.

4. **Operator signature is non-repudiable.** If the operator claims it did NOT mint envelope E, but E carries the operator's public key and verifies, the operator is caught in a lie. The format is falsifiable if we can show signature forgery is computationally infeasible under the discrete log assumption.

5. **Refusal floor violations are auditable.** If an operator discloses a refusal-floor-violating predicate (e.g., a medical diagnosis-like Compass predicate), the audit panel and broader community can inspect the predicate ID in the envelope, check it against the refusal floor, and reject the operator's license to call itself "Calm."

The format is **not** falsifiable if:

- Proofs use non-standard, opaque constructions that cannot be independently implemented.
- Selective disclosure is enforced only by convention, not cryptography.
- The format includes numeric similarity scores or "how close to passing" hints (bits-only output per CONCORD §4).

The reference implementation and this spec ensure all these properties hold.

## §11 — Versioning

The wire format version is `"calm-cross-primitive-envelope/v0"`. Versioning follows semantic conventions:

- **v0 is permanent.** Semantics do not change after publication.
- **Additive changes** (new optional fields, new primitive stubs) are allowed within v0; implementations MUST ignore unknown fields.
- **Breaking changes** require a new version (`v1`). Examples:
  - Changing the canonical encoding rules.
  - Removing or reinterpreting a required field.
  - Changing the signature binding semantics.

Implementations MUST reject envelopes with an unknown `wire_version`.
