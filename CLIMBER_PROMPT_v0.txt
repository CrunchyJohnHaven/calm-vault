5. BINDING DISCIPLINE (non-negotiable — every artifact must satisfy)
==============================================================================
(a) Positive-only predicates. No `tribal=true` denunciation bits. Only
    `untribal_signal_present`-style positives. S49 negation-publication ban.
(b) Forbidden categories (S105): political affiliation, religion, race,
    ethnicity, sexual orientation, criminal history, clinical diagnosis, IQ,
    sensitive medical, immigration, pregnancy, protected classes. Never name
    these as predicates. Append-only forbidden list; closed to contraction.
(c) Per-principal calibration. Thresholds derived from the principal's own
    corpus, never from population norms.
(d) Three-valued logic. `unknown` is a first-class output. Never manufacture
    confidence the data does not support (S129).
(e) Independent evaluator path (S107). Public marker vocabularies + public
    scoring rules. No black-box LLM judges as canonical evaluators (LLM-judge
    only when prompt+model+version are pinned + certificate-attested).
(f) Per-counterparty consent (S62). Every disclosure requires the principal's
    chained consent record for the (predicate, counterparty-class) pair.
(g) Right to deny / opt-out / retract. Principal may disable any single
    dimension without losing the others (S67, S117).
(h) Append-only chain. Corrections are `correction` records (E10), never
    in-place rewrites.
(i) Two-handshake composition (S71). Calm Pact (directive equality between
    agents) precedes Calm Witness (user-state disclosure). Failure of either
    aborts cleanly with zero info exchanged.
(j) Cross-doc consistency. Before claiming alignment between spec docs, verify
    by reading. Path B (CALM_PACT v0.1 = Ristretto255 everywhere) is the ratified
    crypto lock. Old MODP-14 docs carry "superseded" notices.

9. HOW TO MAKE INCREMENTAL PROGRESS ON HARD SUMMITS
==============================================================================
You will not fully bag a Category A-G summit in one session. Aim for one
incremental contribution + a clean handoff. Patterns:

(1) ZK construction: produce a 1-2 page formal sketch — relation,
    public/private witness, prover steps, verifier steps, Fiat-Shamir transcript,
    soundness intuition (not full proof). Mark PARTIAL BAG. Reference E58 for
    the full soundness writeup.

(2) Empirical: produce the experimental protocol + synthetic-data pilot.
    Mark PARTIAL BAG — data collection pending.

(3) Standards: produce the next-N-sections draft, mark PARTIAL BAG —
    section M of total-N.

(4) Governance: produce the constitutional draft + nominee criteria. Mark
    PARTIAL BAG — board not yet seated.

(5) Hardware: produce the integration spec + emulator. Mark PARTIAL BAG —
    physical device test pending.

(6) PQ: produce the candidate-comparison analysis. Mark PARTIAL BAG —
    final selection pending.

(7) Legal: produce the placeholder memo with explicit counsel-review-needed
    flag.

For every partial bag:
  - Write the artifact to a sensible path under ~/AllData/calm_vault_market/<subdir>/.
  - Sister-cite the bagged summits it depends on.
  - End with a "Handoff" section listing exactly what remains and who/what
    can pick it up next.
  - Optionally append a kind:"summit_partial" record to ~/.calm-vault/user_state.jsonl
    per the existing schema; the schema's max summit_number=100 means sub-IDs
    like S118 currently can't chain-record cleanly. Skip chain-write if blocked;
    surface to the schema-extension queue.

10. PARALLELISM RULES
==============================================================================
- Spawn sub-agents on sonnet (workhorse). Opus per-session quota is small.
  Haiku occasionally refuses meta-context prompts.
- Each spawn: focused brief, unique output path, terse `BAGGED Sxxx <path>`
  or `PARTIAL Sxxx <path>` return, do-not-echo-the-doc.
- $20+ per pass budget. ~30 sonnet spawns at ~$0.10 each = ~$3. Spend more,
  spawn more. Saturate the pipeline.
- Use unique file paths to avoid collisions with the parallel CALM session.
  When two sessions race, the one with research/external context yields
  chain-appends to the one without; the meta-substrate-reviewer lane catches
  drift via file mtime checks.
